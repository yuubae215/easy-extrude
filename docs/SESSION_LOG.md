- **2026-04-04**: Feature — Snap nearest-candidate hover highlight: `MeshView.showSnapNearest(position, type)` adds a 17 px / 0.85-opacity intermediate indicator (between candidate 12 px and locked 20 px) for the snap point nearest the cursor within 60 px. `_findNearestSnapCandidate()` in AppController applies back-face culling and drives the indicator in grab, measure, and faceExtrude modes. No-ops added to MeasureLineView and CoordinateFrameView. FaceExtrude snap candidates now also pass through `_filterNearbySnapTargets` (back-face cull + depth filter) consistent with other modes. Feature — SnapPoint UX: (A) Back-face culling — `collectSnapTargets()` now attaches `normal` to face snap targets via `computeOutwardFaceNormal`; `_filterNearbySnapTargets()` (display) and new `_pickBestSnapTarget()` helper (picking) both reject face snap candidates whose normal points away from the camera. (D) Front-facing priority — `_pickBestSnapTarget()` applies up to 5px screen-distance discount to face candidates whose normal directly faces the camera; used in `_measurePickPoint`, `_trySnapFaceExtrude`, and `_trySnapToGeometry`. Bugfix — Rect selection broken when Urban entities present. `_finalizeRectSelection()` used `obj.meshView.cuboid.visible` without optional chaining; `UrbanPolyline`, `UrbanPolygon`, `UrbanMarker` (added 2026-04-01) return `null` for `.cuboid`, causing a `TypeError` that silently aborted the entire handler. Fixed with `cuboid?.visible`. Added "Rect Selection Null Cuboid Guard" rule to CODE_CONTRACTS. Feature — `CoordinateFrameView.updateScale(camera, renderer)` scales the frame group each animation frame so the axis length appears at a constant ~80 screen pixels regardless of camera distance, using the same perspective-correction formula as `MeasureLineView._scaleDots()`.
