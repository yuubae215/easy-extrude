# PostFX & Scene — 仕上げの視覚設計

シェーダーが正しくても、シーンと後処理が素朴だと台無しになる。3D と 2Dシェーダー共通。

## 1. 3色カラーシステム

背景 / 実体(base) / 発光(glow)。発光色は高彩度、背景は黒でなく「ほぼ黒の寒色/暖色」。

| プリセット | 背景 | base | glow | 雰囲気 |
|---|---|---|---|---|
| Cyber Blue | `#070a13` | `#1e293b` | `#3b82f6` | SF標準、外さない |
| Toxic Grid | `#050d08` | `#14261c` | `#22c55e` | ハッカー、マトリックス |
| Synthwave | `#12081a` | `#2a1a3a` | `#e83d84` | レトロフューチャー |
| Amber Protocol | `#120c04` | `#2d2412` | `#f59e0b` | 軍用HUD、警告 |
| Ghost White | `#0a0c10` | `#334155` | `#e2e8f0` | ミニマル、ホログラム |
| Ember | `#140806` | `#3a1d14` | `#f97316` | 灼熱、召喚魔法 |

glow はUI(ボタン・ゲージ・影)、リムライト、ダスト、シェーダー uniform の全てに
同一の `THREE.Color` / CSS変数を共有させて連動させる。ユーザー変更可能にするのが標準。

## 2. アトモスフィア・メニュー (シーンドレッシング)

**同じ装置を毎回使わないこと。** フォグ+グリッド床は「サイバー/テック」1ジャンルの
装置にすぎない。無思考にデフォルト化すると全作品が同じテイストになる。
作品ごとに、題材の意味 (プロダクト紹介? 魔法? 高級品?) に合わせて以下から選ぶ。
直前の案件と同じ組合せは意図的に避ける。

### A. Void Grid — サイバー/テック (SF文脈で明示的に合う時のみ)

```javascript
scene.fog = new THREE.FogExp2(bg, 0.08);
scene.add(gridHelper(20, 40, '#1e293b', '#0f172a'));   // y = -1.5
// + 浮遊ダスト200点 + 三点照明 (白キー0.8 / glow色リム0.5 / Ambient 0.15)
```

### B. Studio — プロダクトショット (SaaS/EC/家電)

フォグなし・床なし。背景は大きなラジアルグラデーション
(ShaderMaterialの背景板 or 内向き大球体: 中心 `#1a2030` → 周縁 `#070a13`)。
接地感は**擬似コンタクトシャドウ**: 物体直下に黒→透明の放射グラデを描いた
PlaneGeometry (CanvasTexture)。照明はソフトな三点+天面の面光源風
(DirectionalLight を弱く複数)。清潔で高級。

### C. Horizon — 地平線と鏡面 (synthwave/夕景/エモーショナル)

背景は上下2色グラデの空 (背景板シェーダー: `mix(topColor, horizonColor, pow(1.0-vUv.y, 2.0))`)。
床は**鏡面リフレクション** (y反転コピー + opacity 0.15–0.3 + 下方向フェード)。
太陽的キーライトを地平線近くの低角度に。フォグは使っても薄い線形フォグのみ。

### D. Nebula — 幻想/宇宙 (魔法召喚・音楽ビジュアル)

shader-2d.md §4 の fbm オーロラを背景板として敷く (uTime ドリフト)。
星: 遠方に小さな Points 300点。床なし・フォグなし、暗部はビネットで締める。
物体は浮遊させ、glow を2色 (`mix(glowA, glowB, noise)`) にすると星雲と馴染む。

### E. Blueprint — 明背景・製図 (ダーク一辺倒への解毒剤)

背景 `#eef2f7` (紙色)。線とワイヤーフレームが主役、実体色は濃紺 `#1e3a5f`、
glow は濃色アクセント (`#2563eb`)。影は薄青。発光は控えめにし、代わりに
**線の描画順・太さの変化**で演出する。UIもライトモードに反転すること。

### F. Abyss — 漆黒ミニマル (高級品・ティザー)

純黒背景 + SpotLight 1灯のみ (角度狭め、penumbra 0.5)。強いビネット。
ダストなし。物体の見え隠れ自体が演出になる。リムライトを背後に1灯だけ足すと
輪郭が闇から浮く。過剰な発光は禁止 — 抑制が高級感。

### 選定原則

- 題材の意味 → 雰囲気 → メニュー選択、の順で決める (装置から入らない)
- ユーザーが世界観を指定したらメニュー外でも新規に設計してよい (このメニューは
  出発点であって檻ではない)
- 1作品内でもフェーズで切り替えられる (例: Abyss で始まり実体化完了で Horizon が明ける)

## 3. トーンマッピングと発光

`renderer.toneMapping = THREE.ACESFilmicToneMapping` を必ず有効化。
本スキルの発光係数 (glow × 1.5〜3.0) はこれが前提。無効だと白飛びする。

### Bloom (使える環境)

EffectComposer + UnrealBloomPass: `strength 0.5–0.8, radius 0.4, threshold 0.85`。
strength 1.0 超は素人っぽく滲むので禁止。

### フェイクブルーム (CDN r128 の three.min.js のみ等)

①発光部の輝度を2.5–3.0倍 ②フレネルリム加算 ③AdditiveBlending のダスト/スプライトを
発光源近傍に配置。この3点で「滲んで見える」状況を作る。本スキルのGLSL係数は
フェイク前提で調整済み。

## 4. フィルミックな画面全体後処理 (2Dシェーダーで自作可)

フルスクリーン板ポリ最終パスに以下を薄く重ねると一気に「作品」になる:

```glsl
vec3 col = texture2D(uScene, vUv).rgb;
// ビネット
float vig = smoothstep(0.9, 0.3, length(vUv - 0.5));
col *= 0.4 + 0.6 * vig;
// フィルムグレイン (毎フレーム変化)
col += (hash21(vUv * 1000.0 + uTime) - 0.5) * 0.05;
// 色収差 (端ほどRGBサンプル位置をずらす) — 演出強調時のみ強く
vec2 dir = vUv - 0.5;
col.r = texture2D(uScene, vUv + dir * 0.004 * uAberration).r;
col.b = texture2D(uScene, vUv - dir * 0.004 * uAberration).b;
// 走査線 (サイバー系のみ)
col *= 0.96 + 0.04 * sin(vUv.y * uResolution.y * 3.14159);
```

`uAberration` は通常 0.3、衝撃の瞬間 (実体化完了・被弾) に 2.0 へスパイク→減衰させる
のが「効果音の視覚版」。グレインは 0.03–0.06 の範囲厳守 (それ以上はノイズ画質に見える)。

## 5. UIクローム (デモページ規約)

- UIの明暗はアトモスフィアに連動させる (A/C/D/F はダーク: slate-900/950基調、
  border slate-800。B はダーク/ライト可、E Blueprint は必ずライト)。角丸 lg–xl。
- 数値・進行度は `font-mono`、ラベルは `uppercase tracking-wider text-[10px]`。
- 必須: エフェクト切替 / 進行度スライダー+再生停止 / グラデーションゲージ /
  base・glow カラーピッカー / 選択エフェクトの実装原理を説明する解説パネル。
- 見出しは `bg-clip-text` グラデーション、キャンバス上オーバーレイは
  `backdrop-blur` + 半透明。
- モバイル: キャンバス `h-[45vh] min-h-[320px]`、パネル縦積み、スライダー thumb 16px+。
