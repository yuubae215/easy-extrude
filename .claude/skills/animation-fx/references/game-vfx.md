# Game-feel VFX — ゲームエンジン風ヒットエフェクト

打撃・斬撃・炸裂・被弾など「ヒットの気持ちよさ」を作る領域。Web演出と文法が違う:
**50–400msの世界**で、時間そのものの操作 (ヒットストップ) と画面全体の反応
(シェイク/フラッシュ/ズーム) を組み合わせる。ボタンの攻撃感、ガチャ演出、
通知の強打、Webゲームに直接使える。

## 1. インパクトの解剖学 (1発 = 6層の合成)

```
0ms      ①インパクトフレーム: 全画面フラッシュ 1–3フレーム (alpha≤0.3、即減衰)
0ms      ②ヒットストップ: 世界の時間を 50–140ms 停止 (会心ほど長い)
0–150ms  ③コア: 白い発光オーブが小→大→消 (中心は必ず白、色は縁に)
0–400ms  ④衝撃波: リングが outExpo で拡大、線幅とalphaは縮退。二重にすると豪華
0–500ms  ⑤破片: スパーク(速度方向に伸びる線) + デブリ(回転する欠片、重力あり)
50ms〜   ⑥画面反応: スクリーンシェイク + ズームパンチ + ダメージ数字
```

順番が本質: フラッシュと停止が「先」、破片と揺れが「後」。逆にすると軽くなる。

## 1b. 余韻 (残し) と うねり

エフェクトの気持ちよさは消え際で決まる — 風呂場で手を叩いた「パチィィィン」の
残響と同じで、**発生は速く、消滅はゆっくり波打ちながら**が原則 (タメツメ)。

- **余韻の基本**: 粒子は初速を高く、強い空気抵抗で急減速させ、そのあと長めの
  尾を引かせる (寿命の後半40–55%が余韻ゾーン)。
- **うねり (動きのノイズ)**: 余韻ゾーンに入った粒子にだけ、連続ノイズ由来の
  漂いを加える。減速だけの余韻より動きの情報量が増え、目を引きつける。

```javascript
const lp = q.life / q.max;                    // 1→0
if (lp < 0.55){                               // 消え際にだけ
  const w = (0.55 - lp) * 620;                // 消えるほど強く漂う
  q.vx += Math.sin(t*7.3 + q.seed*9.0) * w * dt;
  q.vy += Math.cos(t*6.1 + q.seed*7.0) * w * dt * 0.8;
}
```

なんでもかんでもうねらせない — 主役級の粒子 (スパーク・魔力の粒) に限定し、
デブリのような「物理の破片」には加えない (物理が嘘になる)。

## 1c. 構成のタメツメ (リズムと〆)

同じ素材でも構成のリズムで気持ちよさが変わる。原則:
**等リズムの連打 → 一拍の「間」 → 強い一発 (〆)**。

```
0.00-0.36s  小衝撃波×5を等間隔 (90ms) で連打  ← リズムを刻む
0.36-0.62s  何も出さない                      ← 「間」。この静寂が〆を効かせる
0.62s       最大フラッシュ + 長ヒットストップ + 大衝撃波 + 会心数字  ← 〆
```

リズム波形で考える: 一定の波が続いた後、少し休符を置いてから大きく跳ねる。
「間」は 150–300ms — 短すぎると連打に埋もれ、長すぎると別のエフェクトに見える。
実装は予約キュー (`queue.push({at, fn})`) で。連打ビート中にはヒットストップを
入れない (キューの進行が止まりリズムが崩れる)。〆にだけ入れる。

## 2. 時間の操作

### ヒットストップ

```javascript
let hitStop = 0;
function stop(ms){ hitStop = Math.max(hitStop, ms/1000); }
// ループ内: 世界のdtだけ止め、減衰・入力・シェイクは実時間で動かす
let dt; if (hitStop > 0) { hitStop -= realDt; dt = 0; } else dt = realDt;
```

目安: 弱攻撃 40–60ms / 強攻撃 80–100ms / 会心 120–160ms。
「一瞬固まる」ことで脳が重さを補完する。ゲームフィールの最重要テクニック。

### シーケンス予約

収束→炸裂のような多段演出はイベントキューで:
`queue.push({at:0.3, fn:burst})` → ループ内で `at -= dt` (dtなので停止中は進まない)。

## 3. 画面全体の反応

### スクリーンシェイク (trauma方式)

```javascript
trauma = Math.min(trauma + hitPower, 1);      // 加算で受ける
trauma = Math.max(0, trauma - realDt * 2.0);  // 実時間で線形減衰
const sh = trauma * trauma;                   // 二乗: 小揺れは繊細、大揺れは暴力的
const sx = (Math.sin(seed*1.3)+Math.sin(seed*2.7)) * 7 * sh;  // 非整数比サイン合成
ctx.translate(sx, sy); ctx.rotate(Math.sin(seed*1.1) * 0.012 * sh);  // 微回転が効く
```

`Math.random()` の毎フレーム抽選はビリつく。連続ノイズ (サイン合成/simplex) を使う。

### ズームパンチ

`punch = 1.04–1.07` を即時セット → `punch += (1-punch)*(1-Math.pow(0.0005, dt))` で
指数復帰。`ctx.translate(W/2,H/2); scale(punch); translate(-W/2,-H/2)`。

### インパクトフレーム

全画面を白 (または効果色) で1–3フレーム。**光感受性対策で alpha 上限 0.3、
点滅は3回/秒以下、`prefers-reduced-motion` ではフラッシュ・シェイク・
ヒットストップを全て無効化** (パーティクルは残してよい)。

## 4. 部品の作り方 (Canvas 2D)

- **加算合成必須**: 光り物は `globalCompositeOperation='lighter'`。数字・UIは
  source-over に戻す。
- **残像**: 全消去せず `rgba(bg, 0.3–0.4)` の薄塗りクリア。スパークの軌跡が勝手に出る。
- **スパーク**: 点でなく「前位置→現位置の線分」を描く (`moveTo(x-vx*0.03, ...)`).
  太さと alpha は寿命比の1.5乗で減衰。空気抵抗は `v *= pow(0.35, dt)`。
- **斬撃アーク**: 外弧+内弧(逆順)で三日月パスを作りラジアルグラデ塗り
  (内側透明→色→外側透明)。白エッジラインを1本重ねると刃になる。
  2–3枚を**角度と40–80msの時間差**をつけて連撃に。
- **衝撃波リング**: `r = R * outExpo(p)`、`lineWidth = w*(1-p)+0.5`、
  `alpha = (1-p)^2`。太い遅いリング+細い速いリングの二重が定番。
- **デブリ**: 正方形/三角を回転させながら重力落下。数は 12–20 で十分。
- **ダメージ数字**: outBack でポップ (最初の25%で完了) → outCubic で上昇 →
  終盤35%でフェード。会心は 1.7倍サイズ + 別色 + "CRITICAL!" 添字。
  黒縁 strokeText を先に描くと背景を選ばない。

## 4b. エフェクト設計の語彙 (エミッター / パーティクル分類)

ゲームエフェクト業界の標準分解。この語彙で設計すると構成が濁らない:

- **エミッター** = どこから・いつ・どれだけ出すか。
  位置 / 形状 (点・矩形・球・円環・**モデル表面**) / 放出 (量・時間・間隔)。
  すべてのパラメータは **「値 = 固定値 + ランダム幅」** で持つ — この式が
  リアルタイムエフェクトの「毎回少し違う」生命感の正体。
  エミッター自体を移動させながら放出すると軌跡系の表現になる。
- **パーティクル3分類**:
  - **クアッド** (ビルボード): 常にカメラを向く四角。最頻出。Canvas 2Dの円描画も実質これ
  - **モデル**: 3D形状の粒子。爆発の破片・ガレキ (= debris)
  - **ストライプ** (リボン/ポリライン): 四角を帯状に連結。剣の残影・光の閃光・テールランプ
- **アニメーション3手法**: UVスクロール (絵柄だけ流す) / テクスチャアニメーション
  (フリップブック: 稲妻・煙などスプライトシートのコマ送り) / ファンクションカーブ
  (値を任意カーブで制御 = イージングの一般化)。

### ストライプ (リボントレイル) の実装

エミッターの位置履歴を N=12–20 点保持し、帯として描く:

```javascript
// Canvas 2D: 履歴を先細りポリラインで
history.unshift({x, y}); history.length = Math.min(history.length, 16);
for (let i = 1; i < history.length; i++){
  const t = i / history.length;
  ctx.strokeStyle = color; ctx.globalAlpha = (1-t) * 0.8;
  ctx.lineWidth = baseW * (1-t);
  ctx.beginPath(); ctx.moveTo(history[i-1].x, history[i-1].y);
  ctx.lineTo(history[i].x, history[i].y); ctx.stroke();
}
```

three.js では履歴点から三角形ストリップを毎フレーム再構築 (左右に法線方向 ±幅/2)。
剣の残影は「エミッターを剣先にアタッチして振る」だけで成立する。

**剣の残影の実戦則** (ポインタ=剣先のインタラクティブ実装):
- 履歴は「経過時間 (0.2–0.25s)」と「点数上限 (~26)」の両方で刈る。時間だけだと
  低フレームで途切れ、点数だけだと低速で短くなる。
- 二層描画: 外層 = 効果色を太く (alpha低め)、内層 = 白コアを細く。刃に見える。
- **太さと明るさを剣速に比例**させる (`1 + min(speed/2200, 1.6)`)。剣速は
  `speed = speed*0.55 + 瞬間速度*0.45` で平滑化し、入力停止時は指数減衰。
- スラッシュ判定 = 速度閾値 (~2400px/s) + クールダウン (~220ms)。発火で三日月
  アークを移動方向 `atan2(dy,dx)` に向けて出す。
- コンボ検知で構成のタメツメを自動化: 3連スラッシュ → **間 (280ms、発火抑制)**
  → フィニッシャー。間の最中は新規スラッシュ演出を抑制して静寂を守る。
- 初回操作までは**ゴースト入力** (リサージュ軌道) で手本を再生し、実入力で解除。
  モバイルの「触る前に良さが分かる」対策。

## 5. 色の原則

**中心は白、彩度は縁へ**。実際の発光体は中心が飽和して白くなる —
コアオーブ・スパークの根元・数字は白系、効果色 (炎=amber, 雷=cyan,
魔法=fuchsia, 会心=red/yellow) はグラデの外周とリングに置く。
全体を効果色で塗ると安っぽくなる。

## 6. 実装の規律

- パーティクル総数に上限 (CAP 500–800)。超過分はスポーンしない (古いのを消さない —
  消すと発生中のエフェクトが欠ける)。
- 連打は仕様。すべての発動関数は「途中から重ねて呼ばれる」前提で書く
  (グローバル状態は Math.max / 加算で受ける)。
- ヒットの手応え = 停止+揺れ+フラッシュの「画面反応」が7割。パーティクルを増やす
  前に②⑥を調整する。
- UIへの転用: ボタン押下に stop(30) + 数字ポップだけでも「打鍵感」が出る。
  通知の強調はシェイクでなく**その要素だけ**の translate 振動 (全画面を揺らさない)。

## 7. スポーン/消滅への転用

出現・消滅も「ヒット」として設計すると締まる。共通レシピ:
**発動フレームのフラッシュ + 微シェイク + 床リング + 光柱**を、実体化エフェクト
(three-3d.md: dissolve / shatter 等) に重ねる。消滅は加速系イージング (inOutSine/inCubic)
で「砕ける」、スポーンは減速系 (outCubic/outExpo) で「定着する」— 非対称が物理感の核。
uProgress を対称に設計しておけば (SKILL.md Step 3)、shatter の逆再生のように
1つの実装で双方向が手に入る。

## 8. スクロール/通常UIとの合流 (STRIKE SCROLL パターン)

ゲームの手応えをLPやアプリに移植する際の作法:

- **ヒットストップは使わない**。スクロールという「ユーザーが動かしている世界」を
  止めると酔う・苛立つ。停止の代わりに要素側のスラム
  (`scale 0.955 → 1.012 → 1` + 輝度フラッシュ、outExpo 500ms) で重さを出す。
- **trauma上限を下げる** (0.3前後、加算上限0.6)。シェイクはページラッパーの
  transform にだけかけ、FXオーバーレイ (fixed canvas, pointer-events:none) は揺らさない。
- 発動は IntersectionObserver (threshold 0.4–0.5)、1要素1回。Replay時は
  dataset フラグと struck クラスを外し unobserve→再observe。
- 全画面フラッシュ alpha は通常の半分以下 (≤0.12)。閲覧の邪魔をしない強度が「合流」。
- 着弾点は要素の視覚的重心 (左上寄り、`left + min(width*0.35, 260px)` 程度) に置くと
  テキストと衝突しない。
